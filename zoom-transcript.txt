### Zoom Accounts and Licenses: Structure

#### 1. **Owner/Admin Account (Your Main Account)**
- This is the primary Zoom account for your organization (e.g., “ABC Institute of Education”).
- You purchase all licenses from this account.
- Has full administrative (and billing) control.

#### 2. **Licensed Users**
- Each Zoom “license” grants full Pro/Business/Enterprise meeting capabilities to a single user account.
- You **assign** each license to a user under your organization via Zoom's admin portal.
- Users can be real people (e.g., teachers), or can be generic, such as `host1@abc-institute.org`, `host2@abc-institute.org`.

#### 3. **User Management**
- All these user accounts are “members” of the main account and centrally managed by the owner/admin.
- You can create these accounts and assign licenses from your Zoom admin dashboard.

#### 4. **How Scheduling Works**
- When you use the Zoom API to schedule a meeting, you pick which licensed user (host) the meeting is scheduled for using their User ID or email.
- **Each licensed user can host one concurrent meeting** at a time.

#### 5. **Branding & Control**
- By controlling the profile settings of each user (host), you can set their display name to “ABC Institute of Education” or similar, so your brand shows up, not the teacher’s personal name.

3. **You send the ‘start_url’ to the designated teacher for that session (or use it yourself as admin).**
4. **The teacher/admin clicks the ‘start_url’ and joins the meeting as host, without knowing or needing the Zoom account credentials.**
